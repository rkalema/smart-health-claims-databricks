# Data Dictionary

## Catalog & Schema Configuration
- **Catalog:** `health_claims_dev` (Unity Catalog)
- **Schemas:** `bronze`, `silver`, `gold`
- **Storage Location:** `dbfs:/mnt/data/health_claims/` (configured via `dbutils.fs.mount`)

---

## Bronze Layer Tables

### bronze.claims_raw
Raw claims data ingested from source systems via Auto Loader.

| Column | Type | Description |
|--------|------|-------------|
| claim_id | STRING | Unique claim identifier |
| patient_id | STRING | Patient identifier (PII - to be masked in Silver) |
| incident_type | STRING | Type of incident (Fall, Motor Vehicle, etc.) |
| patient_reported_severity | STRING | Severity reported by patient |
| provider_requested_amount | DECIMAL | Amount requested by provider |
| clean_claim_date | DATE | Date claim was processed |
| avg_heart_rate | DECIMAL | Average heart rate from RPM devices |
| avg_sleep_score | DECIMAL | Average sleep quality score |
| plan_type | STRING | Insurance plan type |
| ingestion_timestamp | TIMESTAMP | Auto-generated by Auto Loader |

### bronze.ehr_cdc_raw

| Column | Type | Description |

| event_type | STRING | INSERT, UPDATE, or DELETE |

| timestamp | STRING | Event timestamp |

| ingestion_timestamp | TIMESTAMP | Auto-generated by Auto Loader |

### bronze.medical_images_raw
Medical imaging metadata.

| Column | Type | Description |
|--------|------|-------------|
| file_path | STRING | Path to image file |

## Silver Layer Tables

### silver.claims_clean
Cleaned and validated claims data with PHI masked.

| Column | Type | Description |
|--------|------|-------------|
| claim_id | STRING | Unique claim identifier |
| patient_id_hash | STRING | SHA-256 hash of patient_id |
| incident_type | STRING | Type of incident |
| patient_reported_severity | STRING | Severity level |
| provider_requested_amount | DECIMAL | Requested amount |
| clean_claim_date | DATE | Claim date |
| avg_heart_rate | DECIMAL | Average heart rate |
| avg_sleep_score | DECIMAL | Average sleep score |
| plan_type | STRING | Plan type |
| claim_amount | DECIMAL | Validated claim amount |
| claim_date | DATE | Validated claim date |
| last_updated | TIMESTAMP | Last update timestamp |

**Data Quality Rules Applied (EXPECT CONSTRAINTS):**
- `CONSTRAINT valid_claim_id EXPECT (claim_id IS NOT NULL)`
- `CONSTRAINT valid_amount EXPECT (provider_requested_amount > 0)`
- `CONSTRAINT valid_incident EXPECT (incident_type IN ('Fall', 'Motor Vehicle', 'Sports Injury', 'Workplace'))`
- `CONSTRAINT valid_severity EXPECT (patient_reported_severity IN ('Minor', 'Moderate', 'Severe'))`
- `CONSTRAINT valid_plan EXPECT (plan_type IN ('Basic', 'Standard', 'Premium'))`
- `CONSTRAINT valid_date EXPECT (clean_claim_date <= CURRENT_DATE())`

## Derived Fields & Business Logic

### Calculated Metrics
- **patient_id_hash**: `sha2(patient_id, 256)` for PHI protection.
- **claim_amount**: Cast to `DoubleType` from `provider_requested_amount`.
- **claim_date**: Cast to `DateType` from `clean_claim_date`.

### Business Rules
- **High Value Claim**: `provider_requested_amount > 50000`
- **Requires Manual Review**: `amount > 100000 OR incident_type IN ('Workplace', 'Motor Vehicle') OR severity = 'Severe'`
- **Allowed Amount Calculation**: 
  ```sql
  CASE 
    WHEN plan_type = 'Premium' THEN billed_amount * 0.90
    WHEN plan_type = 'Standard' THEN billed_amount * 0.80
    WHEN plan_type = 'Basic' THEN billed_amount * 0.70
  END
